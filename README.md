# ホテル客室清掃・運用管理システム (Hotel Room Ops Management System)

リアルタイムで客室の清掃ステータス、スタッフ割り当て、破損・忘れ物報告を管理できる、ホテル運用のための統合管理システムです。
清掃スタッフ（クリーナー）と管理者（マネージャー）間の緊密なコミュニケーションと迅速なステータス同期を、Firebase Firestore を用いて実現しています。

---

## システムの特徴と主要機能

### 1. 管理者ポータル (Manager Portal)
- **リアルタイム客室ダッシュボード**: 
  - 全客室の状態（未清掃・清掃中・確認中・清掃完了）の一括監視。
  - フロアごとのフィルタリング、清掃スタッフの動的な割り当て。
- **通知・報告管理**: 
  - クリーナーから報告された「破損」「忘れ物」データを写真証拠付きで閲覧・管理。
  - 報告データの個別確認消去・一括一掃機能。
- **客室・スタッフ設定**: 
  - 客室を新規追加するフォーム、削除機能。
  - 清掃スタッフの追加・ロール管理。
- **清掃管理基準（チェックリスト）設定**:
  - 全客室で共有される清掃項目（チェックリスト）の追加、編集、削除（グローバル管理）。

### 2. 清掃スタッフポータル (Cleaner Portal)
- **タスク一覧**: 
  - ログインしたスタッフに割り当てられた、または未割り当ての客室リストを直感的に表示。
- **詳細チェックリスト**: 
  - 管理者が定義した項目に沿って、タップで進捗をチェック。
- **報告送信ツール**: 
  - 「破損」「忘れ物」に分類し、現場でカメラを起動して撮影した写真や、写真をアップロードして管理者へ即時送信するフォーム。

### 3. 超高速なリアルタイム同期 (Real-time DB Sync)
- **Firebase Firestore Integration**: 
  - 現場スタッフがチェックリストを更新、または報告を送信すると、管理者のダッシュボードへミリ秒単位で反映。リフレッシュなしでチーム全体の状況を認識できます。

## デザイン言語とUI/UX設計 (Design Philosophy)

本システムは **「スイス・プロフェッショナル」** の設計哲学に基づいて構築されています。現場での過酷なマルチタスク環境に最適化されています。

- **フォント構成 (Typography)**:
  - **ディスプレイ・数字用**: `Space Grotesk` (技術的で、識別性の高いフォント。客室番号や統計数値に採用)
  - **本文・UI用**: `Inter` (ニュートラルで超高精細なサンセリフ。細かなメモやステータスも読みやすい)
- **カラーパレット (Tailwind CSS)**:
  - 管理者/ベース: `slate-900` (#0f172a, 重厚感と信頼性)
  - 清掃中ステータス: `blue-600` (#2563eb, 視認性の高い青)
  - 警告/問題報告: `rose-600` (#e11d48, 注意を惹く鮮やかな赤)
  - 完了ステータス: `emerald-700` (#047857)
- **細部のつくり込み (Micro-interactions)**:
  - `motion` を使用したスムーズなタブ切り替え、ポータルの役割ごとのスライド、モーダルの展開。
  - タッチターゲットは最小 44px 以上、指一本で快適に操作できるサイズ。
  - 誤操作を防ぐために「削除」「一掃」ボタンには2ステップの確認UIをインラインで配置。

## 🛠️ 技術スタック (Tech Stack)

- **フロントエンド**: React 19 (TypeScript), Vite 6
- **アニメーション**: `motion/react`
- **スタイリング**: Tailwind CSS v4 (ユーティリティクラスによるモダンスタイリング)
- **インフラ/DB**: Firebase Firestore, Firebase Storage, Firebase Auth
- **アイコン**: `lucide-react` (統一されたベクターアイコンセット)

## 開発環境のセットアップと起動方法

### 1. 依存関係のインストール
プロジェクトのルートディレクトリで以下のコマンドを実行し、パッケージをインストールします。

```bash
npm install
```

### 2. 環境変数の設定
ローカル環境で使用する場合は、`.env` ファイルを作成し、以下のFirebase設定情報を記述してください。  
（※ `.env.example` を参考にしてください。）

```env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

### 3. 開発サーバーの起動
Vite 開発サーバーを起動します（本環境はポート `3000` で稼働します）。

```bash
npm run dev
```

### 4. ビルドとプロダクション起動
本番環境向けの静的ファイルのビルド、または動作確認を行います。

```bash
# ビルド
npm run build

# ローカルプレビュー
npm run preview
```
